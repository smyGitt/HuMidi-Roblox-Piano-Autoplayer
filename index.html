MIDI Performance System

Humidify parses standard MIDI files and executes them via simulated keyboard inputs. It utilizes stochastic timing variations, heuristic hand-assignment, and dynamic articulation algorithms to emulate human performance characteristics, bypassing the rigid quantization typical of standard macro engines.

Core Architecture & Subsystems

Algorithmic Humanization Engine

Introduces customizable stochastic variations to note start times, durations (articulation), and tempo. This includes a simulated hand-drift parameter that utilizes a decay factor to correct temporal drift over time, preventing long-term desynchronization across the track.

Heuristic Fingering Assignment

Analyzes pitch values and temporal proximity to dynamically assign notes to simulated left or right hand clusters. This enables realistic chord-rolling execution based on sequential note offsets rather than simultaneous block execution.

Dynamic Pedal Actuation

Parses harmonic structures and bass note intervals to automatically generate sustain pedal inputs. The subsystem utilizes bounding parameters to detect unsafe intervals (such as minor seconds or tritones) and forces a pedal reset to avoid dissonant audio overlap.

Real-Time Graphical Interface

Renders an active PyQt6 piano visualizer and scrubbing timeline. This allows the user to monitor execution states, active pitches, and measure boundaries synchronized directly to the parsed MIDI tempo map.

Configurable Error Generation

An optional execution subsystem that probabilistically introduces adjacent key strikes. When triggered, it calculates a logical mistake pitch (prioritizing adjacent diatonic keys) to simulate physiological performance fatigue.

Zero Local Dependencies

Deployed as a standalone Windows Portable Executable (PE) utilizing PyInstaller. The Python host process and required dynamically loaded libraries (DLLs) are bundled within the binary payload.

Execution Parameters

The initialization sequence requires explicit parameter configuration prior to programmatic execution.

  • 01

    Parsing & Track Selection

    The application utilizes the mido library to parse the MIDI structure. Users manually select specific tracks or isolate percussion channels via the PyQt6 interface.

  • 02

    Parameter Configuration

    Users define the limits of the stochastic variables, including timing variance (in seconds), articulation overlap (as a percentage), and global tempo scaling.

  • 03

    Process Injection

    Upon triggering the pynput global hotkey, the application compiles the runtime events into a priority queue and begins sending synthesized keystrokes to the active operating system window manager.
